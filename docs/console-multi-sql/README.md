# 控制台多 SQL 执行

## 执行范围

- 编辑器存在非空选区时执行选中内容；没有选区时执行全部内容。
- 无论来源是选区还是全文，都会使用 `splitSql` 按语句边界拆分，因此选区内也可以包含多条 SQL。
- `splitSql` 不会在字符串、反引号标识符、行注释或块注释内部按分号拆分。
- 状态栏回显实际执行的当前结果 SQL，便于确认执行范围。

## 执行与结果

- Archery `/query/` 一次只处理一条 SQL，前端按原顺序逐条请求。
- 某条 SQL 失败时记录错误并继续后续语句，不把单条失败伪装成整批成功。
- 每条语句生成一个结果页签，展示成功/失败状态、行数和耗时。
- 执行过程中结果逐条出现，默认激活最后一个结果；切换页签时网格、状态栏和 CSV 导出同步切换。
- 查询历史按拆分后的单条 SQL 记录，并保存执行成功状态、实例、数据库、模式和数据库类型。

## 编辑体验

- SQL 草稿按环境保存，关闭控制台后重新打开可恢复内容和实例/数据库/schema 上下文。
- 美化功能按语句分别格式化，再用分号和换行连接，支持同一编辑器中的多条 SQL。
- 自动联想只分析光标所在语句，不会被控制台中的其他 SQL 引用表干扰。

## 核心模块

| 模块 | 职责 |
|------|------|
| `sql-editor.mjs` | `splitSql`、`extractSqlStatementAt`、选区解析和多语句美化 |
| `console-draft.mjs` | 控制台草稿防抖保存与恢复 |
| `app.js` | 顺序执行、结果状态和历史写入 |
| `app-events.mjs` | 结果页签、执行快捷键和导出事件 |

## 验证

- 单元测试覆盖引号、注释、转义、尾部分号、空语句、光标所在语句和选区解析。
- Playwright E2E 覆盖选区执行、全文多语句、结果页签切换、失败继续和历史记录粒度。
- Windows E2E 覆盖选区执行、全文多语句、结果页签切换、失败继续和历史记录粒度；当前结果以测试输出为准。
