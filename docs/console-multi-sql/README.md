# 控制台多 SQL 执行

## 执行范围

- 编辑器存在非空选区时执行选中内容；没有选区时执行全部内容。
- 无论来源是选区还是全文，都会使用 `splitSql` 按语句边界拆分，因此选区内也可以包含多条 SQL。
- `splitSql` 不会在字符串、标识符、行注释、嵌套块注释或 PostgreSQL dollar-quote 内部按分号拆分。
- 状态栏回显实际执行的当前结果 SQL，便于确认执行范围。

## 执行与结果

- Archery `/query/` 一次只处理一条 SQL，前端按原顺序逐条请求。
- 某条 SQL 失败时记录错误并继续后续语句，不把单条失败伪装成整批成功。
- 每条语句生成一个结果页签，展示成功/失败状态、行数和耗时。
- 执行过程中结果逐条出现，默认激活最后一个结果；切换页签时网格、状态栏和 CSV 导出同步切换。
- 查询历史按拆分后的单条 SQL 记录，并保存执行成功状态、实例、数据库、模式和数据库类型。
- 普通 SELECT 默认每页 100 条，并额外执行一次最多统计 1000 条的 COUNT；界面展示本页条数和分页，不展示总条数。
- 每个结果页签独立翻页；翻页和切换每页条数只重查当前语句，不重复写入查询历史。
- 数据修改 CTE、锁定 SELECT、`SELECT INTO`、DML/DDL 和已含顶层分页参数的 SQL 不自动分页。
- CSV 会重新执行有界 COUNT，最多导出当前查询的前 1000 条，而不是只导出当前页。

## 编辑体验

- 标签栏最左侧固定控制台入口可新建、打开默认控制台或从“所有”列表切换。
- 每个环境会保存全部控制台，以及各自名称、SQL、实例、数据库、schema、编辑器高度和打开状态；主动关闭的控制台不显示为标签，但仍会出现在“所有”列表中并可重新打开。两级列表均支持重命名，删除图标用于永久删除对应控制台记录。
- 美化功能按语句分别格式化，再用分号和换行连接，支持同一编辑器中的多条 SQL。
- 自动联想只分析光标所在语句，不会被控制台中的其他 SQL 引用表干扰。

## 核心模块

| 模块 | 职责 |
|------|------|
| `sql-editor.mjs` | `splitSql`、`extractSqlStatementAt`、选区解析和多语句美化 |
| `console-session.mjs` | 多控制台会话校验、防抖保存、旧草稿迁移与恢复 |
| `console-workspace.mjs` | 控制台命名、运行态创建与持久状态转换 |
| `console-menu-view.mjs` | 固定入口、默认控制台和全部控制台菜单 |
| `console-rename.mjs` | 控制台重命名校验、弹窗交互和持久化触发 |
| `console-query.mjs` | 安全查询分类、分页 SQL 与 COUNT SQL |
| `console-execution.mjs` | 首次查询、COUNT、翻页和执行上下文快照 |
| `console-result-view.mjs` | 结果网格和分页栏渲染 |
| `app.js` | 顺序执行、结果状态和历史写入 |
| `app-events.mjs` | 结果页签、执行快捷键和导出事件 |

## 验证

- 单元测试覆盖引号、注释、转义、尾部分号、空语句、光标所在语句和选区解析。
- Playwright E2E 覆盖选区执行、全文多语句、控制台列表切换、重载恢复、结果页签切换、失败继续和历史记录粒度。
- Windows E2E 覆盖控制台工作区持久化与上述真实应用流程；当前结果以测试输出为准。
