# 数据网格与资源树交互

## 数据网格

- 表数据和控制台结果共用网格渲染模块，列宽按内容估算并设置上限。
- 表头右侧支持拖拽调整列宽，宽度在当前标签页内保持。
- 表数据支持多字段组合排序：升序 → 降序 → 取消，工具栏显示并可单独移除排序条件。
- 表数据分页展示精确总条数、总页数和本页条数，并按页码边界控制首页、上一页、下一页和末页按钮。
- 单元格点击后可用 `Ctrl+C` 复制完整值；`NULL` 复制为空字符串。
- 表数据与控制台结果均支持 CSV 导出。

每次加载或刷新表数据时都会额外执行一条 `COUNT(*)`，复用当前表和 WHERE 条件，但不包含排序、`LIMIT` 或 `OFFSET`。大表、缺少合适索引或条件复杂时，COUNT 可能带来额外数据库负载；总数查询失败会显式展示错误，不把 `/query/` 的 `affected_rows` 当作总条数。

## 筛选与表浏览

- WHERE 工具栏接受条件片段，例如 `status = 1 AND name LIKE 'A%'`，并提供当前表字段、关键词和数据库方言函数联想。
- 单击表节点直接打开数据浏览；点击左侧箭头展开列、键和索引。
- 表和列的悬浮提示展示类型与注释；表结构按需请求并缓存。
- MySQL 资源层级为“实例 → 数据库 → 表”，PostgreSQL 为“实例 → 数据库 → 模式 → 表”。
- MySQL 结构视图从 `CREATE TABLE` 解析字符串、数字、`NULL` 和 `CURRENT_TIMESTAMP[(精度)]` 等默认值；DDL 高亮将反引号/双引号标识符作为原子 token，避免 HTML 污染。

## 树与控制台联动

- 控制台切换实例、数据库或 PostgreSQL 模式时，左侧树自动展开并高亮对应节点。
- 点击树上的数据库或模式节点时，当前控制台同步切换上下文。
- 新建控制台继承当前表标签或最近一次树选择的实例、数据库和模式。
- 跨 schema 同名表使用独立标签和元数据上下文，不会互相覆盖。

## 代码结构

| 模块 | 职责 |
|------|------|
| `grid.mjs` | 网格 HTML、列宽计算、转义 |
| `resource-tree-view.mjs` | MySQL/PostgreSQL 资源树渲染 |
| `ddl.js` / `table-view.mjs` | MySQL CREATE TABLE / PostgreSQL 列元数据解析，以及数据、结构和 DDL 子视图 |
| `app-events.mjs` | 点击、输入、快捷键、拖拽和悬浮事件绑定 |
| `db-context.mjs` | 数据库类型、schema 上下文、分页查询和 COUNT SQL 生成 |

SQL 自动联想的完整规则见 [SQL 自动联想增强](../sql-autocomplete-enhancements/)。

## 验证

- 自动化验证与手工验证应覆盖空表、总数恰好为页大小整数倍、筛选后总数变化、末页边界和 PostgreSQL schema COUNT。
- `test/e2e.js` 提供 MySQL 表数据/结构/DDL、PostgreSQL schema 表数据/结构、排序筛选相关基础链路。
