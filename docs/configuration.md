# 配置与本地数据

## 环境配置规则

- `id` 必须在本地唯一，建议只使用字母、数字、短横线和下划线。
- `scheme` 只接受 `http` 或 `https`。
- `base` 可包含端口，但不能包含协议、路径、空白、查询参数或片段。
- 默认环境文件只在本地没有任何环境配置时生效。

不要在仓库、Issue、PR、截图或日志中提交真实环境地址、账号或凭据。

## Windows 本地数据

非敏感数据位于：

```text
%APPDATA%\com.fanxiaofan.sql-studio\store.json
```

其中包括环境、用户名、记住密码标志、查询历史和控制台工作区会话。会话保存全部控制台的 SQL、数据库上下文和打开状态，关闭标签不会删除记录；查询结果不保存。密码单独保存在 Windows 凭据管理器，服务名为 `sql-studio`，账号使用环境 `id`。

轻量模式相关键也由 Rust 原生层维护：`lightweight_mode` 控制冷启动是否创建主 WebView，`ui_snapshot` 保存侧边栏、活动标签和最近上下文等重建界面所需元数据。原生窗口位置和尺寸由 Tauri window-state 文件单独保存。前端销毁后，这些文件和 Rust 运行时状态仍然存在。

完整清理：

1. 退出 SQL Studio；
2. 删除 `%APPDATA%\com.fanxiaofan.sql-studio\`；
3. 打开 Windows“凭据管理器”，删除与 SQL Studio 环境对应的通用凭据；
4. 如需彻底移除程序，再删除便携 exe。

仅删除 exe 不会自动删除配置和凭据。

## 备份与迁移

- 可以备份 `store.json` 或手工记录环境字段，但文件可能包含查询历史和内部结构信息，应按敏感数据处理。
- Windows 凭据不会随 `store.json` 迁移，换机后需要重新输入密码。
- MCP Access Token 保存在 Windows 凭据管理器中，正常重启不会变化；只有在 MCP 管理界面主动重置时才会更新。
- 当前没有正式的配置导入/导出协议；跨版本或跨设备直接复制内部存储结构前，应先核对数据格式变化。
